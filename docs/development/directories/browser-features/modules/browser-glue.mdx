## Overview

Floorp extends the Firefox browser process through an ECMAScript module named `BrowserGlue.sys.mts`. This module acts as a central hub for parent-process initialization and is responsible for registering custom JavaScript Window Actors. These actors facilitate structured message passing between the parent process and child processes, enabling Floorp-specific features to interact with browser chrome and content.

## Window Actor Registration

The module registers Floorp-specific window actors using the `ActorManagerParent.addJSWindowActors` API. According to the source, 22 actors are registered within this file (`browser-features/modules/modules/BrowserGlue.sys.mts`).

## Loader Integration

The bridge loader imports `BrowserGlue` to initialize internationalization, record loader state, filter enabled modules, and run module lifecycle hooks (`bridge/loader-features/loader/index.ts`). This integration ensures that the parent process is prepared for the subsequent loading of Chrome UI features and pages.

## Architecture Context

In the Floorp architecture, the ESM Modules layer sits above the patched Gecko base and provides the foundation for the Bridge and Chrome UI layers (`browser-features/modules/modules/BrowserGlue.sys.mts`). By managing Window Actor registration here, Floorp establishes a robust communication channel that the Bridge layer can rely on when loading features dynamically.
